# #책스타그램 인기 게시물 30개 분석

**작성일**: 2026-05-09
**상태**: ⏸️ **V1 이후로 연기 (2026-05-09 결정)** — 자동 수집은 인스타 API/스크래핑 차단으로 불가, 수동 캡처는 V1 출시 후 실제 사용자 데이터로 대체 검증하기로 함. 본 프레임워크는 V1.5+ 비주얼 정합성 점검 시점에 재가동.
**소요 시간**: 60–90분 (캡처 30분 + 채우기 30분 + 합성 30분)
**목표**: 5개 카드 템플릿이 실제 #책스타그램 인기 결과 분포에 맞는지 검증, 빠진 결이 있는지 발견

## 1. 데이터 수집 (30분)

### 1.1 어디서

**인스타그램 앱**에서 다음 해시태그를 검색하고 **인기 탭**(좋아요 많은 순)으로 정렬:

- `#책스타그램` (메인, ~50만 게시물 추정)
- `#책귀절` `#책귀절스타그램` (인용구 특화)
- `#독서스타그램`
- `#bookstagram` (영문 비교용 — 영어권 결 참고)

### 1.2 수집 규칙

- **인기 탭 상위 30개**를 순서대로 캡처 또는 URL 메모
- 광고·이벤트·서평단 게시물은 제외 (자연 발생 콘텐츠만)
- 같은 계정 게시물이 연속 등장하면 **계정당 최대 2개**까지만
- 표지 사진만 있고 인용구 텍스트 없는 게시물은 제외 (우리 앱 결과물과 비교 대상이 아님)

### 1.3 메모 방법

다음 중 편한 방법:
- (A) **Notion DB 또는 Google Sheets**에 복붙 — 권장
- (B) **인스타 스크린샷** + 본 파일에 직접 표 채우기 — 빠르지만 회고 어려움
- (C) **인스타 저장(즐겨찾기) 기능**으로 30개 모은 뒤 한 번에 분석 — 가장 빠름

## 2. 분석 스키마 — 9개 차원

각 게시물마다 아래 9개 컬럼을 채웁니다.

| # | 컬럼 | 값 후보 | 비고 |
|---|------|---------|------|
| 1 | **post_id** | URL 또는 일련번호 | 추적용 |
| 2 | **type** | `single` / `carousel` / `story_capture` | 캐러셀이면 첫 장 기준 |
| 3 | **layout** | `text_only` / `text_with_cover` / `photo_overlay` / `design_template` / `mixed_collage` | 아래 정의 |
| 4 | **font_style** | `serif` / `sans` / `handwritten` / `mixed` / `unclear` | 인용구 텍스트 폰트 |
| 5 | **color_mood** | `light` / `dark` / `warm_neutral` / `pastel` / `high_contrast` / `cover_extracted` | 전체 분위기 |
| 6 | **composition** | `centered` / `asymmetric` / `magazine` / `poetry` | 텍스트 배치 |
| 7 | **quote_length** | `short ≤50` / `mid 50–200` / `long 200+` | 한국어 글자 수 기준 |
| 8 | **book_genre** | `essay` / `novel` / `self_help` / `poetry` / `nonfiction` / `unclear` | 책 분류 |
| 9 | **app_origin** | `app_visible` / `app_likely` / `manual` / `unclear` | 워터마크·결로 추정 |

### 2.1 layout 정의

- **text_only**: 인용구 텍스트만 있고 이미지·표지 없음 (T5 타이포에 해당)
- **text_with_cover**: 인용구 텍스트 + 책 표지가 같은 카드 안에 (T1·T2)
- **photo_overlay**: 사진 위에 인용구 텍스트가 얹혀짐 (T4 표지 발췌·또는 풍경 사진)
- **design_template**: 도형·일러스트·강한 디자인 요소가 있는 카드 (앱 생성물 가능성↑)
- **mixed_collage**: 여러 요소 콜라주, Tezza·Canva 스타일

### 2.2 app_origin 정의

워터마크나 디자인 결로 추정:
- **app_visible**: "Tezza" "Canva" 등 워터마크 명시
- **app_likely**: 명시는 없지만 디자인 일관성·정밀도가 명백히 앱 생성물
- **manual**: Adobe·Figma·종이 사진·손글씨 등 직접 만든 결
- **unclear**: 판단 어려움

## 4. 합성 — 표 다 채운 후 답할 질문 7개

### Q1. layout 분포는?

5개 layout 값별 카운트를 계산:
```
text_only:        ___개 / 30
text_with_cover:  ___개 / 30
photo_overlay:    ___개 / 30
design_template:  ___개 / 30
mixed_collage:    ___개 / 30
```

**의미**:
- `text_with_cover`가 다수 (>40%) → T1·T2 우선순위 정확
- `text_only`가 의외로 많음 (>30%) → T5 타이포는 길이 제약 풀고 V1 정식 템플릿으로
- `design_template`이 많음 → 앱 생성물에 거부감 적음, T4 같은 적극 디자인도 OK
- `photo_overlay`가 많음 → T4 표지 발췌가 핵심 차별화 자리, 정밀도 더 투입

### Q2. font_style 분포는?

```
serif:        ___개
sans:         ___개
handwritten:  ___개
mixed:        ___개
```

**의미**: 우리 토큰의 `Noto Serif KR` 선택이 시장과 정합인지. handwritten이 의외로 많으면 V2 폰트 추가 후보.

### Q3. color_mood 분포는?

`warm_neutral`이 많으면 우리 Ink-Paper-Copper 시스템이 시장 결과 일치. `cover_extracted`가 많으면 T4의 가치 검증.

### Q4. quote_length 분포는?

```
short ≤50:   ___개
mid 50–200:  ___개
long 200+:   ___개
```

**의미**: T5 타이포(≤50자 전용)의 적용 비율. long 200+이 적으면 자동 폰트 축소 알고리즘은 일반 케이스 위주 최적화로 충분.

### Q5. book_genre 분포는?

`essay` 또는 `self_help`가 dominant → 알라딘 검색 첫 노출 카테고리 우선순위.

### Q6. app_origin 분포는?

```
app_visible:  ___개
app_likely:   ___개
manual:       ___개
```

**의미**:
- `manual`이 50% 이상 → 시장이 "앱 생성물 티"에 거부감 강함, 워터마크 최소 노출 결정 정당화
- `app_visible`이 30% 이상 → 사용자들이 앱 사용에 거부감 적음, 워터마크 더 적극적 가능
- 어느 앱이 자주 보이는지 기록 (Tezza? Canva? 한국 앱?) → 직접 경쟁자 식별

### Q7. 우리 5개 템플릿이 커버하지 못하는 결이 있는가?

데이터에서 발견했지만 우리 템플릿에 없는 결이 있다면:
- (예시) 손글씨 폰트 + 종이 결 (handwritten, warm_neutral)
- (예시) 풍경 사진 위 인용구 (photo_overlay이지만 표지가 아닌 자연·여행 사진)
- (예시) 만화·일러스트 캐릭터와 함께 (design_template 중 일부)

발견된 결이 인기 게시물의 20% 이상을 차지한다면 → V2 템플릿 후보로 기록.

## 7. 시간 절약 팁

- **표지 사진만 있는 게시물은 빠르게 스킵** — type 컬럼만 채우고 다음으로
- **layout이 같은 게시물 연속 5개 보이면 같은 결로 묶음** — 일일이 채우지 말고 묶음 단위로
- **app_origin은 모를 때 `unclear` 적극 사용** — 추측하다 분석 늘어지지 말 것
- **30개 다 못 채워도 OK** — 20개부터 패턴이 보이기 시작. 25개 채우면 합성 가능
