## 0. 한 줄 결론

> **북모리(Bookmory)가 V1 컨셉의 80%를 이미 제공하고 있다.** "OCR + 인용구 + SNS 카드"만으로는 차별화 부족. 살아남으려면 ① 단톡방 친구 단위 메커닉 ② 표지 컬러 자동 팔레트 ③ 데이터 주권(웹·내보내기) ④ AI 인용구 어시스트 중 **2–3개를 V1에 박아야 한다.**

## 1. 요약 (tl;dr)

- **결정적 발견**: 한국 1인 개발 앱 **북모리** — OCR + 인용구 노트 + SNS 카드 변환 + 알라딘 책 정보 + 감성 디자인 + Flutter 개발 + 평점 4.9. V1 컨셉과 거의 동일. 기존 평가 리스트에서 누락돼 있었다.
- **누락 경쟁자 7개 추가 식별**: 북모리, 북트리, 북노트, 북플립, 리더스, 그믐, 플라이북 AI. 글로벌 트렌드 갱신: Fable "Read in Social Mode", StoryGraph mood 통계.
- **PLAN.md의 기존 차별화 축 5개 중 3개 약화**:
  - "모바일 퍼스트" — 북모리도 모바일 퍼스트
  - "인스타그래머블 카드" — 북모리·북트리·Tezza 모두 카드 제공
  - "한국 도서 DB" — 북모리·북적북적·플라이북 모두 한국 도서 풍부
  - "친구 발견" — 약화되지 않음 (대부분 솔로 도구)
  - "V2 AI" — 약화되지 않음 (한국 인용구 AI는 빈자리)
- **살아있는 빈자리**: ① 친구 단톡방 단위 메커닉 ② 표지 컬러 → 카드 팔레트 자동 ③ 데이터 내보내기·웹 ④ 인용구 특화 AI ⑤ "감성 무드" 컬렉션.
- **권장 V1 코어 축 재정의**: "모바일 + 카드 디자인 + 한국 도서"에서 → **"단톡방 친구 + 표지 자동 팔레트 + 데이터 주권"** 으로 좁힘.

## 2. 방법론과 한계

### 2.1 데이터 출처

이번 조사는 **외부 데스크 리서치**다:
- Google Play Store / App Store 공식 페이지
- 한국 언론·블로그 리뷰 (덴 매거진, 캠퍼스플러스, 데일리팝, longblack.co 등)
- 클리앙·디시인사이드 등 사용자 커뮤니티 리뷰
- 글로벌 비교 (Book Riot, BookBub, peaceincolor 등)

### 2.2 한계 (먼저 짚고 가기)

**할 수 있는 것**:
- 경쟁 제품의 명목 기능·UI 분포·시장 평판 파악
- 시장 공백·기회 가설 도출
- PLAN.md 차별화 축의 stress-test

**할 수 없는 것**:
- 실제 retention·DAU·매출 수치 (비공개)
- UX 마찰의 깊이 (직접 사용해야 함)
- 한국 사용자가 갈아탈 임계점 (실제 인터뷰 필요)

**따라서 본 문서의 용도**: V1 차별화 결정을 **외부 자료까지 본 상태에서** 내릴 수 있게 하는 것. 본인이 추후 북모리·Tezza·리더스를 직접 1주씩 사용하면 본 문서의 가설을 확정·반박할 수 있다.

### 2.3 후속 검증 권장 (본인이 직접)

- [ ] **북모리 1주 사용** ★★★ — 본 문서의 핵심 가정 모두 직접 확인
- [ ] **리더스 5일** — "인스타 결 책 SNS" 실제 활성도
- [ ] **북적북적 3일** — 1M+ 유저 메커닉(책탑 시각화)이 retention에 어떻게 작동하는지
- [ ] **Tezza 3일** — 카드 1장 만드는 데 정확히 몇 분?
- [ ] **Fable / StoryGraph 각 3일** — 글로벌 차별화 메커닉 직접 경험

## 3. 갱신된 경쟁사 지도

### 3.1 분류 축

- **솔로 도구** vs **소셜 SNS** vs **유틸리티**
- **한국** vs **글로벌**

### 3.2 한국 시장 (7개)

| # | 제품 | 분류 | 핵심 가치 | 강점 | 약점 (우리 기회) | 우리와 거리 |
|---|---|---|---|---|---|---|
| 1 | **북모리** ★★★ | 솔로 도구 + 감성 | OCR + 인용구 + SNS 카드 + 알라딘 | 평점 4.9, Flutter, 1인 개발 깊이, OCR 작동 | 광고 빈도 ↑, PC 없음, 내보내기 약함, 책 정보 미흡, 소셜 부재 | **거의 동일 (충돌 80%)** |
| 2 | 북적북적 | 솔로 도구 + 시각화 | 책을 탑으로 쌓아 시각화 | 1M+ 유저, #북적북적 8.6만 게시물, retention 메커닉 강함 | 인용구·카드 약함, 디자인 귀여움 결로 책스타·디자이너 타겟 아님 | 다른 결 (시각화 vs 인용구) |
| 3 | 북트리 | 솔로 도구 + 유틸 | 도서 관리 + 독서노트 + 인용구 폰트 | 폰트 선택 다양, SNS 공유 | UI 구식, 모던하지 않음 | 우리와 결 비슷, 디자인 차로 우위 가능 |
| 4 | 북노트 | 솔로 도구 | 독서 기록 + 책 추천 | 사용자 풀 적음 | 활성도 낮은 듯 | 약한 경쟁자 |
| 5 | 북플립 | 솔로 도구 | 독서 차트·달력·통계 | 통계·시각화 체계적 | 인용구·소셜 약함 | 다른 결 |
| 6 | **리더스** | 소셜 SNS | 인스타 결 + 자체 북클럽 미션 | 북클럽 차별화, 팔로워 메커닉 | 사용자 적음, 인용구 카드 약함 | **친구 발견 축에서 직접 충돌** |
| 7 | **그믐** | 소셜 + 북클럽 | 온라인 북클럽 토론 | 깊이 있는 토론 커뮤니티 | 무거움(가입 마찰), 모바일 카드 약함 | 다른 결 (라이트 vs 헤비) |

**그 외 한국 메이저 (전자책·오디오북, 직접 경쟁 아님)**:
- 밀리의서재 (전자책 구독, 20만 권)
- 리디 (전자책·웹툰)
- 윌라 (오디오북, 9,900원/월)

→ 이들은 **콘텐츠 플랫폼**, 우리는 **메타·인용구**. 충돌보다는 **연동 후보 (V2/V3)**.

**한국 AI 책 도구**:
- **플라이북 AI** — 성별·연령·기분·관심사·독서 기록 분석해 책 추천 + 책 내용 요약
- **교보문고 AI 책봇** — 시범 서비스, 추천 → 판매 검증 단계
- 두 곳 모두 **책 추천**에 집중. **인용구 특화 AI는 한국에 없음** → 빈자리.

### 3.3 글로벌 (5개)

| # | 제품 | 분류 | 핵심 가치 | 한국화 가능한 메커닉 |
|---|---|---|---|---|

## 4. ★ 북모리 심층 분석 (가장 직접적 경쟁자)

### 4.1 무엇을 제공하는가

공개 마케팅·리뷰에 따르면:
- 책 검색 + 알라딘 책 정보 자동 매칭
- 독서 타이머 + 캘린더 + 통계
- **인용구 노트** — 직접 입력 + **사진→OCR 텍스트 추출**
- **인용구 → SNS 업로드용 이미지 변환**
- 감성 디자인 (1인 개발자가 디자인까지)
- Flutter로 개발 (우리와 동일 스택)
- **유료 버전**: 광고 제거, 더 많은 통계, OCR 무제한 (무료는 보상형 광고 시청 후 OCR 사용)
- 평점 4.9 (Play Store)
- iOS 출시 2020 이후 꾸준히 업데이트

### 4.2 강점 — 우리가 배워야 할 것

1. **1인 개발자도 평점 4.9** — UI/UX 디테일에 시간 충분히 투자한 결과. 우리도 카드 5종 디테일 절대 타협하면 안 됨.
2. **OCR이 메인 도입 동선** — 사진 한 장으로 인용구가 추출되는 경험이 핵심 wow moment. 우리도 OCR 작동 안정성이 V1 최소 요구.
3. **알라딘 책 정보 자동** — 기본기. 우리는 이미 구현됨 (Stage 1 완료).
4. **광고 + 구독** 비즈 모델로 1인 개발자 생계 가능. V2 freemium 4,900원 정책의 검증.

### 4.3 약점 — 우리가 메울 빈자리

(클리앙·DC 리뷰 + Play Store 사용자 후기 기반)

1. **광고 빈도 ↑** — "뭐만 하려 하면 광고가 떠서 갈아탔다"는 후기. → **우리는 V1에 카드 생성 광고 0** 정책으로 직격.
2. **PC 버전 없음, 데이터 내보내기 약함** — "블로그에 옮겨담으려는데 복사가 어렵다". → **우리는 웹 뷰어 + Notion/Markdown export**가 hook.
3. **구독만, 1회 구매 옵션 없음** — 사용자 불만 명시. → V2 freemium 설계 시 lifetime 옵션 고려.
4. **책 정보가 제대로 안 나오는 경우 있음** — 알라딘 매칭 실패 케이스. → 우리는 ISBN 바코드 스캔 + 표지 사진 검색 + 직접 추가 fallback 3중화.
5. **소셜·친구 없음** — 100% 솔로 도구. → **친구 단톡방 단위 메커닉이 직접 빈자리.**
6. **귀여움 결 디자인** — 디자이너 페르소나(한지영, 9,900원 결제의향)는 안 씀. → **Ink-Paper-Copper 모던 결**로 디자이너·책스타그래머 세그먼트 흡수.

### 4.4 우리 V1과 직접 충돌하는 5가지 지점

| 충돌 지점 | 북모리 | 우리 V1 계획 | 결론 |
|---|---|---|---|
| OCR 인용구 추출 | ✅ ML Kit 추정 | ✅ ML Kit | **동일 — 차별화 X** |
| 인스타용 카드 | ✅ 변환 기능 | ✅ 5종 템플릿 | **디자인 품질로만 우위 가능** |
| 알라딘 책 정보 | ✅ | ✅ Edge Function 프록시 | **동일 — 차별화 X** |
| 감성 디자인 | ✅ 귀여움 결 | ✅ Ink-Paper-Copper 모던 결 | **세그먼트 분리** (북모리=일반/우리=책스타·디자이너) |
| 1인 개발자 | ✅ | ✅ | **동일 — 차별화 X** |

→ **"오! 책 인용구 모으고 카드 만들기" 한 줄 가치 제안만으로는 북모리에 밀린다.** 차별화 축을 추가해야 한다.

### 4.5 컨셉이 살려면 필요한 차별화 축 (필수 1+ 강함)

- (a) **단톡방 친구 단위 인프라**가 핵심 — 북모리는 100% 솔로
- (b) **표지 컬러 → 카드 팔레트 자동** 이 디자이너 세그먼트 결정타
- (c) **데이터 주권(웹 + 내보내기)** 은 북모리 약점 직격
- (d) **인용구 특화 AI** 는 한국 빈자리

위 (a)~(d) 중 **2개 이상 V1에 박지 않으면 북모리 vs 책귀 비교 시 "북모리가 먼저 나왔고 동일"** 으로 결론날 가능성이 높다.

## 5. 한국 시장 다른 플레이어 (요약)

### 5.1 북적북적 — 1M+ 유저의 시각화 메커닉

- **메커닉**: 책을 탑으로 쌓는 시각화. 페이지·두께 반영. 캐릭터 보상.
- **시사점**: **"시각적으로 쌓이는 retention 메커닉"** 의 위력. #북적북적 8.6만 게시물 = 홈 화면 캡처가 SNS에 자연스럽게 올라감.
- **우리 입장**: 인용구가 "쌓이는 비주얼" — 책별 인용구 컬렉션이 자라는 그래프·서가 비주얼 메커닉을 V1.5에 검토.

### 5.2 리더스 — 친구 발견 + 북클럽 미션

- **메커닉**: 인스타 결 SNS + 자체 운영 북클럽(미션 + 다른 독자 소통).
- **시사점**: **북클럽**이 cold start 방어책. 우리도 "친구 0명" 일 때 채울 콘텐츠 필요.
- **우리 입장**: V1은 솔로 도구 + 단톡방 공유로 시작. V1.5에 "주간 인용구 챌린지"(앱이 주제 제시 → 모두가 인용구 1개씩) 검토.

### 5.3 그믐 — 헤비 북클럽

- **메커닉**: 정기 북클럽 + 토론. 가입·운영 마찰 크다.
- **시사점**: 책 좋아하는 사람이 진지한 토론 욕구 있음. 그러나 V1 타겟(연 5–20권, 인스타 활성)에는 무거움. **다른 세그먼트**.

### 5.4 북트리·북노트·북플립

- 북트리 — 인용구 폰트 선택, SNS 공유. UI 구식. 디자인 우위만 잡으면 됨.
- 북노트 — 사용자 풀 적음.
- 북플립 — 통계 강함. 우리는 통계는 V1.5 이후.

### 5.5 플라이북 AI / 교보문고 AI 책봇

- 둘 다 **책 추천** AI. **인용구 처리 AI는 없음.**
- **빈자리**: "이 인용구를 다른 인용구와 묶기", "이 책 읽은 사람들이 좋아한 인용구", "이 인용구 영어 번역" 같은 **인용구 특화 AI** — 한국에 아직 없음.

## 6. 글로벌 영감 — 한국화 가능 메커닉

### 6.1 Fable — "Read in Social Mode" ★★

- **메커닉**: 친구·북클럽 멤버가 같은 책을 읽으며 **하이라이트·반응을 실시간 공유**. 챕터별 spoiler-free 룸.
- **우리에게**: **친한 친구 3–5명 단위 "동시 독서"** 결을 단톡방 결과 결합. 페르소나 3(이수연)이 이미 하는 행동의 인프라화.
- **차이**: Fable은 자체 ebook 리더. 우리는 종이책·전자책 모두 OK (인용구 단위 공유).

### 6.2 StoryGraph — Mood + 통계 + Shareable Graphs

- **메커닉**: 책마다 mood/pace/페이지 트래킹. 연말 리포트 등 shareable 통계 그래프.
- **우리에게**: **"무드별 인용구"** — "위로 / 새벽 3시 / 출근길 / 이별" 등 태그. 인용구를 무드별로 다시 묶어 보여줌 = "사진 찍지만 다시 안 봄" 페인 해결의 답.

### 6.3 BookTok / 책스타그램

- **메커닉**: 인플루언서가 책 판매 견인. 2024 BookTok 5,900만 부.
- **우리에게**: 출시 직후 **책스타그래머 5–10명 베타 + 카드 템플릿 추천 의뢰** 채널이 viral 기점. PLAN.md에 이미 명시되어 있으나 **인플루언서 친화 export 포맷**(워터마크 위치 옵션, 캐러셀 자동 생성)을 V1에 박을지 검토.

### 6.4 Readwise — 다시 보기 푸시

- **메커닉**: 저장한 인용구를 푸시·이메일로 다시 보여줌.
- **우리에게**: PLAN.md V2 항목. **변경 없음** — V1에 끼면 좋지만 비용 큼.

### 6.5 Goodreads / Letterboxd

- 둘 다 **친구 timeline + 평점**. 우리는 평점 안 다룸 (인용구만). PLAN.md 결과 일치.

## 7. ★ 차별화 가설 7개 — 데이터 기반 평가

각 가설을 **(근거 / V1 비용 / 검증 방법 / V1 vs V1.5 vs V2 / 위험)** 으로 평가.

### H1. 단톡방 친구 N명 인프라화 ★★★

- **가설**: 친구 follow가 아니라 **기존 단톡방을 인용구 챌린지·동시 독서 인프라**로. 단톡방에 카드 보내면 받는 사람이 한 탭으로 "내 서재 추가 + 챌린지 참여" 가능.
- **근거**: 페르소나 3(이수연) 인터뷰에 명시 — "친구 셋이서 매주 한 책 챌린지 단톡방". 북모리·북적북적·리더스 모두 약함.
- **V1 비용**: 중. 카카오 공유 deep link + 챌린지 데이터 모델 (이미 deep-link 인프라 있음, Stage 1).
- **검증**: 실제 인터뷰에서 "최근 단톡방에 책 인용구 보낸 적?" "친구 단위 동시 독서 경험?". 베타 출시 후 K-factor 측정.
- **V1 / V1.5 / V2**: **V1 코어 후보 — 단톡방 공유 1탭은 V1, 챌린지 메커닉은 V1.5.**
- **위험**: 카카오 deep link UX가 OS·버전마다 갈림. PWA·웹 fallback 필수.

### H2. 표지 컬러 → 카드 팔레트 자동 ★★★

- **가설**: 책 표지에서 dominant 컬러 5–7개 추출 → 카드 배경·아랑소·텍스트 색 자동 매칭.
- **근거**: 페르소나 5(한지영) 직접 요청 — "그게 있으면 매일 쓸 것 같다". `palette_generator` 이미 추가됨, `docs/design/color-extraction.md` 명세 있음. 북모리·Tezza 모두 고정 템플릿 색.
- **V1 비용**: 낮음. 이미 인프라 완비, 알고리즘만.
- **검증**: 자체 dogfooding — 책 30권 표지로 팔레트 뽑아서 5개 카드 템플릿에 자동 적용, 본인 인스타에 올렸을 때 자연스러운지.
- **V1 / V1.5 / V2**: **V1 코어 — Stage 3에서 무조건 박는다.**
- **위험**: 흑백·단색 표지에서 팔레트 빈약. fallback으로 디자인 시스템 Ink-Paper-Copper 토큰.

### H3. 인용구 특화 AI (Claude API) ★★

- **가설**: Claude API로 "이 인용구 한 줄 요약", "비슷한 결의 다른 인용구 3개", "이 책의 핵심 문장 3개 자동 추천", "이 인용구 영어 번역" 같은 인용구 특화 AI 어시스트.
- **근거**: 한국 책 AI는 모두 **추천**(플라이북, 교보). **인용구 AI는 빈자리.** 사용자가 이미 Claude 헤비 유저(PLAN.md), API 비용 통제 가능.
- **V1 비용**: 중. Edge Function 1개 추가, 호출당 0.5~2원, 사용자당 일 5회 캡.
- **검증**: 베타 5명에게 "이 기능 없으면 안 쓸 것 같다"인지 평가. retention D7 비교.
- **V1 / V1.5 / V2**: **V1.5 후보** (V1은 안정성 우선, V1.5에 추가). 다만 마케팅 메시지에 V1.5 베타 명시.
- **위험**: 한국어 인용구 짧으면 AI 결과 빈약. 페이지·문맥(앞뒤 문장)도 같이 보내야 품질.

### H4. 데이터 주권 (웹 뷰어 + Markdown/Notion 내보내기) ★★

- **가설**: 모은 인용구를 **언제든 가져갈 수 있다**. Markdown export, Notion 자동 동기화, 웹 뷰어(읽기 전용).
- **근거**: 북모리 약점 직격 — "PC 없어서 갈아탔다", "복사가 어렵다". 디지털 독서가(노션·옵시디언 사용자) 유입 hook.
- **V1 비용**: 낮~중. Markdown export는 즉시 가능. Notion API 연동은 V1.5.
- **검증**: 베타에 Notion 사용자 1–2명 포함시켜 actual 사용 관찰.
- **V1 / V1.5 / V2**: **V1에 Markdown export만, Notion 연동은 V1.5.**
- **위험**: 디자이너 페르소나(한지영)·일반 인스타 사용자(김민지)는 무관심 — 핵심 가치보다 hygiene factor.

### H5. 무드 컬렉션 — "감정 태그 + 다시 보기" ★★

- **가설**: 인용구에 무드 태그 ("위로 / 새벽 3시 / 출근길 / 이별 / 다짐"). 무드별 컬렉션 자동 생성. **"사진 찍지만 다시 안 봄" 페인을 직접 해결.**
- **근거**: 페르소나 1(김민지) "다시 안 봄" 페인. StoryGraph mood 결 검증된 결. Readwise는 다시 보기만, 우리는 **테마 단위** 다시 보기.
- **V1 비용**: 낮. 태그 시스템 (이미 PLAN.md V1 명시).
- **검증**: 자체 dogfooding — 본인이 모은 인용구 무드 태그를 회고했을 때 가치 느끼는지.
- **V1 / V1.5 / V2**: **V1에 태그만, 무드 추천·자동 분류는 H3와 묶어 V1.5.**
- **위험**: 사용자가 태그 안 붙임 — UX에서 카드 만들 때 자연스럽게 1–2개 추천 제시.

### H6. 인플루언서 친화 export ★

- **가설**: 1080×1920 세로 스토리, 1080×1350 캐러셀 4장 자동, 워터마크 위치·크기 옵션, 로고 빼기 옵션 (인플루언서 한정 베타).
- **근거**: BookTok·책스타그래머가 viral 채널 핵심. PLAN.md "Build in public + 책스타그래머 5–10명 컨택" 명시.
- **V1 비용**: 낮. Stage 3 카드 export에서 size variant 추가만.
- **검증**: 책스타그래머 5명 컨택해 "이 export 옵션이 본인 콘텐츠 워크플로에 fit하는지" 직접 인터뷰.
- **V1 / V1.5 / V2**: **V1에 size variant만, 워터마크 빼기는 V1.5+유료.**
- **위험**: 워터마크 빼기 = viral 트래픽 손실. 트레이드오프 명확히.

### H7. 인용구 누적 비주얼 (북적북적 결) ★

- **가설**: 인용구가 책별로 쌓이는 비주얼(서가, 카드 더미, 그래프). 사용자가 홈 화면을 SNS에 올리고 싶게.
- **근거**: 북적북적 #북적북적 8.6만 게시물 — 시각화가 자체 viral. 우리는 인용구로 다른 결의 비주얼.
- **V1 비용**: 중~높음. 디자인+구현 둘 다 무겁다. 디자인 시스템 결과 결합도 고민.
- **검증**: Stage 2 끝나는 시점에 prototype 1장으로 페르소나 1·3에 보여줘서 "이 화면을 인스타에 올리겠는가?" 직접 질문.
- **V1 / V1.5 / V2**: **V1.5 후보** — V1 안정성 흔들지 말 것. V1은 서재 리스트로 충분.
- **위험**: 디자인 까다로움, 캐릭터 결(북적북적)과 차별화 명확히 해야.
