1. 홈 / 피드 탭

북모리=현재 읽는 책 대시보드, 북적북적=책탑 캡처 산출물(친구 피드 없음), Goodreads=친구 타임라인이지만 사실상 죽은 피드(150M 유저에도). → 책귀 V1은 "친구 새 인용"이라는 빈 화면을 쓰지 않고 "내 인용 피드 + 단톡으로 받은 카드 함"으로 출시. cold start 면역 + 쌓이는 게 보임(retention) + 그 화면이 캡처감. follow 타임라인은 V1.5에 같은 피드에 합쳐 진화.

2. 서재 탭

북적북적의 "책탑"은 두께·페이지 비례로 쌓이는 시각화 = 그 화면이 곧 SNS 캡처 산출물(#북적북적 8.6만 게시물). 원리는 훔치되(누적이 시각적으로 보이고, 캡처감일 것) 책더미 자체·귀여운 캐릭터는 안 베낀다(우리 모던 톤과 충돌, 디자이너 세그먼트 이탈). V1은 단순 리스트 유지 + 책 카드에 "이 책에서 모은 N구절" 배지 + 표지 dominant color 띠를 심어두면 V1.5 "인용 서가" 시각화로 저비용 확장.

3. 인용구 입력 화면 /quote/new — 그룹 1 (Stage 2, 최우선)

북모리는 OCR이 메인 도입 동선이지만 무료는 보상형 광고 시청 후(= "뭐만 하려 하면 광고" 이탈 주범) + 페이지·태그가 부가 입력으로 묻힘. → 책귀는 OCR 결과 + 페이지 + 무드 태그 + 노트를 한 화면에서 편집, OCR은 "편집 가능한 초안 채우기"로만(자동 확정 X), 원본 사진 대조 뷰, OCR에 광고 절대 안 붙임. 권한 거부·OCR 실패·검색 0건 어디서든 "직접 입력" 출구 상시 노출.

6. 인용 목록 / 상세 (Readwise 카드 문법 + 무드 인라인)

Readwise 하이라이트 카드 = 표지+제목/저자+텍스트+노트+태그, 다차원 필터, 인라인 태깅(읽다가 T 한 번으로 그 자리에서 태그). 디자인 철학도 동족 — 하이라이트 색이 "물리적 형광펜 모사"(우리 "오래된 잉크·종이"와 같은 사고). → 책귀는 Readwise 카드 구성을 한국어·한국 책 메타로, 무드 태그는 저장 직후 토스트의 "무드 추가?" 칩으로 인라인. 필터 3개로 충분(책별/무드별/최근순) + 상단 검색.
