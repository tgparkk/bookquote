# 경쟁 제품 평가 프레임워크

**용도**: Validation 단계에서 경쟁 제품을 1주씩 직접 사용하며 평가
**목적**: 우리 앱의 차별화 포인트를 데이터 기반으로 좁히기

---

## 평가 대상 (5개)

가상 인터뷰에서 발굴된 신규 경쟁자 **Tezza** 추가됨.

| # | 제품 | 카테고리 | 우선순위 | 평가 기간 |
|---|---|---|---|---|
| 1 | **Goodreads** (모바일) | 글로벌 1위 책 SNS | ★★★ | 1주 |
| 2 | **Readwise** | 인용구 저장·다시 보기 | ★★★ | 1주 |
| 3 | **Letterboxd** | 영화 리뷰 SNS (참고) | ★★ | 5일 |
| 4 | **북적북적** | 한국 책 SNS | ★★ | 5일 |
| 5 | **Tezza** ⭐ | 인스타 카드 만들기 도구 | ★★★ | 1주 |

Tezza는 Goodreads·Readwise와 다른 결의 직접 경쟁자 — "사용자가 카드를 직접 만드는 도구". 우리 앱은 이걸 자동화·통합.

---

## 평가 항목 (체크리스트)

각 제품마다 동일한 항목으로 평가. 점수보다 **구체적 관찰** 우선.

### 1. 첫 인상 (5분)
- [ ] 앱스토어 화면·첫 5초 인상은? (사용자가 보는 첫 광고)
- [ ] 회원가입까지 몇 단계? 어떤 정보 요구?
- [ ] 온보딩 흐름 (튜토리얼)? 좋은지 답답한지?
- [ ] **첫 5분에 가치를 느끼는가?** (Aha moment 시점)

### 2. 핵심 기능 사용 (1일)
실제 책 1권을 골라 다음을 시도:
- [ ] 책 검색·추가 (소요 시간, 한국 도서 검색되는지, ISBN 바코드 스캔 가능한지)
- [ ] 인용구 저장 (직접 입력·OCR·복붙 — 어느 방식 지원?)
- [ ] 인용구 검색·다시 보기
- [ ] 친구·팔로워 찾기·연결
- [ ] 카드 만들기 (해당된다면 — Tezza·Letterboxd)
- [ ] SNS·메신저 공유

### 3. 디자인·UX (3일)
- [ ] 폰트·색·여백 — 모던한가, 답답한가?
- [ ] 한국어 가독성 OK?
- [ ] 모바일 제스처 자연스러운가?
- [ ] 다크 모드 지원?

### 4. 일주일 사용 후 retention 신호
- [ ] 매일 열게 되는가? 며칠 만에 안 열게 됐는가?
- [ ] 어떤 기능을 가장 많이 썼나?
- [ ] 짜증 나는 순간 5개 기록
- [ ] **"내가 만들면 어떻게 다르게 만들까" 5가지**

### 5. 수익화 관찰
- [ ] 무료/유료? 가격? 기능 차이?
- [ ] 광고 노출 빈도·결?
- [ ] 결제 유도 패턴 (paywall 위치)
- [ ] 본인이 결제할 의향 생겼는가? 왜?

### 6. 우리와의 차별화 가능성
- [ ] 이 제품이 **하지 않는 것** 중 우리가 잘할 수 있는 것은?
- [ ] 이 제품이 **잘하는 것** 중 우리가 따라잡기 어려운 것은?
- [ ] 이 제품 사용자 중 누구를 우리가 데려올 수 있을까?

## 제품별 사전 가설 (가상 인터뷰 기반)

실제 사용 후 이 가설이 맞는지 틀렸는지 표시할 것.

### Goodreads
**사전 가설**: UI 구식, 한국 도서 부족, 모바일 최적화 약함, 인용구 카드 기능 부재. 책 메타데이터·평점은 강함.
**확인할 것**: 한국어 검색·UX·신간 반영 속도

### Readwise
**사전 가설**: 인용구 저장·다시 보기 push 알림이 핵심 가치. 디자인은 미니멀하지만 "공유"가 약함. 가격대 높음.
**확인할 것**: 무료 체험 가능 여부, 한국어 인용구 다루는 품질, 가격

### Letterboxd
**사전 가설**: 친구 timeline + 짧은 리뷰 + 평점이 핵심. 우리 앱의 직접적 영감원. 디자인 모던.
**확인할 것**: 사용자가 시간을 어떻게 쓰는지, 어떤 인터랙션이 가장 자주 일어나는지, 카드·이미지 공유 흐름이 있는지

### 북적북적
**사전 가설**: 한국 시장이지만 사용자 적음, UI는 평범, 차별화 약함.
**확인할 것**: 일일 활성 사용자 추정, 어떤 기능이 살아있고 어떤 게 죽었는지

### Tezza ⭐
**사전 가설**: 일반 인스타 카드 만들기 도구. 책 전용 X. 폰트·템플릿 다양. 카드 1장 만드는 데 5–10분 걸림. 책 정보 자동 매칭 X.
**확인할 것**:
- 책 인용구 카드 만드는 데 정확히 몇 분 걸리는가? (시간 측정)
- 폰트·템플릿 몇 종류?
- 결제 모델
- **사용자가 책 카드용으로 쓴다는 증거**가 있는가? (앱 내 검색·해시태그)

## 평가 결과 정리 양식

각 제품마다 다음 형태로 1페이지 정리:

```
# [제품명] 평가 결과

## 한 줄 요약
[핵심 가치 1줄 + 우리에게 시사점 1줄]

## 강점 (우리가 배워야 할 것) — 3–5개
- ...

## 약점 (우리가 메울 빈자리) — 3–5개
- ...

## 차별화 가능성
- 우리가 더 잘할 것:
- 우리가 못 따라잡을 것:

## 인용·기록할 만한 5가지 관찰
1.
2.
3.
4.
5.

## 사전 가설 검증 결과
- 가설 1: [맞음/틀림] — 근거
- ...

## 우리 V1에 즉시 반영할 것
- ...
```
