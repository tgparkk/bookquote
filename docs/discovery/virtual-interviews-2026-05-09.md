**인터뷰어 메모**: "사진 찍지만 다시 안 봄" — 이게 진짜 핵심 페인. 우리 앱이 해결할 수 있는 문제. 그러나 "다시 봐야 할 동기"가 사용자에게 약함 — Readwise식 알림이 V2에 있는 게 다행.

> "(웃음) 솔직히요? 평가하기 전에 제가 직업이 디자인이라 굉장히 깐깐해요. 책 카드 앱 많이 봤거든요. 다 별로였어요. 폰트가 너무 흔하거나, 레이아웃이 어색하거나, 책 표지 색이랑 안 맞거나."

> "딱 한 가지 신경 쓰는 거: 앱이 만들어준 카드라는 게 티 나면 안 돼요. '아 이거 어떤 앱이지?' 이런 게 제일 싫어요. 자연스러워야 해요. 폰트가 5종 미만이면 한계가 와요."

> "그리고 책 표지 컬러를 뽑아서 자동으로 팔레트 만들어주면 그건 진짜 시간 단축이에요. 그게 있으면 매일 쓸 것 같고. 없으면 그냥 일러스트레이터 써요."

> "친구 follow 기능은… 저는 이미 인스타 팔로워가 있어서요. 굳이 또 다른 곳에서? 근데 책 전용 인스타 같은 게 되면 그땐 매력적일 수도. 실제 책 좋아하는 사람만 모인."

### Q5. 가격?

> "디자인이 정말 좋으면 9,900원도 OK. 결과물이 인스타 올릴 수준이면. 13,900원? 그건 좀. 근데 디자인 별로면 1,000원도 비싸요. 어차피 일러스트레이터 있으니까."

**핵심 시그널 (긍정)**:
- 결제 의향 가장 높음 (9,900원)
- 매우 구체적 기능 요청: **"책 표지 자동 컬러 추출 → 팔레트 생성"** ← V1에 거의 무조건 넣어야 할 듯
- 책 카드 = 본인 콘텐츠 → 강한 internal motivation

**핵심 시그널 (위험)**:
- 디자인 품질 임계점이 매우 높음
- "한 권에 인용구 3–5개" — V1에 책당 인용구 개수 보여주기·정렬 UX 필요
- 친구 follow 무관심 — 이미 인스타에 팔로워 있음. SNS 대체가 아니라 보완으로 포지셔닝
- 폰트 다양성이 부족하면 즉시 이탈

# 종합 분석

## 1. 페르소나별 우리 앱 적합도

| 페르소나 | 적합도 | 핵심 가치 |
|---|---|---|
| 1. 김민지 (마케터, 28) | ★★★ 핵심 타겟 | "사진은 찍는데 다시 안 봄" 페인 해결 |
| 2. 박준호 (개발자, 35) | ☆ 외곽 | 안 씀. 마케팅 제외 |
| 3. 이수연 (대학생, 23) | ★★★ 핵심 타겟 | 친구 단톡방 챌린지 + 카드 시간 단축 |
| 4. 정현우 (영업, 42) | ☆ 외곽 | 안 씀. 마케팅 제외 |
| 5. 한지영 (디자이너, 31) | ★★ 까다로운 핵심 | 결제 의향 가장 높음, 디자인 임계점 높음 |

**3명 / 5명**이 강한 긍정 신호 → 약한 PMF 신호. **모두 여성**, **모두 인스타 활성 사용자**, **모두 단톡방·SNS에서 책 공유 행동 이미 존재**.

## 2. 핵심 인사이트 (페르소나 교차 분석)

### 🟢 검증된 것
- **"사진은 찍는데 다시 안 봄"** 페인은 강함 (1, 3 공통)
- **단톡방 책 공유 행동 이미 존재** (1, 3 공통) — 우리는 이걸 인프라화
- **카드 만드는 시간 부담**이 명확한 페인 (1, 3, 5 공통) — Tezza 5분, 일러스트레이터 30분
- **타겟은 인스타 활성 + 책 좋아하는 여성 20–30대** (1, 3, 5)
- **결제 의향 4,900원은 합리적**, 9,900원은 디자인 품질 강한 타겟에서만

### 🟡 주의 신호
- **"친구 timeline" 가치는 페르소나마다 다름** — 핵심 타겟인 페르소나 1조차 미온적, 페르소나 5는 무관심. 페르소나 3만 핵심으로 봄
- → **친구 timeline을 V1의 핵심으로 두는 건 위험**. 솔로 도구로서 가치 + 단톡방 공유 메커닉이 더 안전
- **디자인 임계점 매우 높음** (페르소나 5) — 첫 인상이 별로면 이탈, 회복 어려움
- **"잘난 척 부담"** (페르소나 1) — 앱 자동 생성 카드는 이 부담을 낮춤. 단 워터마크가 너무 두드러지면 반대 효과

### 🔴 발견된 위험
- **Tezza라는 직접 경쟁자 발견** (페르소나 3) — Validation 단계에서 직접 사용해봐야 함. 플랜의 경쟁사 리스트에 추가 필수
- **이미 인스타 팔로워 가진 책스타그래머는 friend follow 무관심** (페르소나 5) — 이들은 우리 앱을 도구로 쓰지 SNS로 쓰지 않음. 이중 포지셔닝 필요

## 3. 기능 우선순위 재조정 (인터뷰 결과 반영)

### V1에 무조건 들어가야 할 것 (인터뷰에서 강한 시그널)
- **책 표지 컬러 자동 추출 → 카드 팔레트 생성** ← 페르소나 5의 가장 구체적 요청
- **카드 생성은 무료** ← 페르소나 1
- **한국어 폰트 5종 이상** ← 페르소나 3, 5 — 부족하면 즉시 이탈
- **책당 인용구 3–10개 보기·정렬** ← 페르소나 5
- **단톡방 공유 1탭** ← 페르소나 1, 3

### V1 격하 또는 V2 이동 검토
- **친구 follow + timeline** — 핵심 타겟 5명 중 1명만 강하게 원함. **솔로 도구 + 단톡방 공유**가 더 안전한 V1 코어. timeline은 V1.5나 V2로
- 위험 회피 안 한다면: timeline은 넣되 비주얼·메인 화면 위계에서 한 단계 내림

### 새로 발견된 기능 후보 (V2+)
- **"이미 형광펜 그은 거 통합"** (페르소나 2 — 비타겟이지만 인사이트) — 리디·밀리·교보 e-book 연동 (V3)
- **인용구 다시 보기 푸시** — 페르소나 1의 "다시 안 봄" 페인 직접 해결. 플랜에 V2로 있는 게 정확

## 4. 인터뷰 가이드 다듬기 (실제 인터뷰 시)

다음 질문을 추가하면 좋을 듯:
- **"카드 만들 때 가장 시간 걸리는 부분은?"** — Tezza 같은 경쟁 도구의 약점 파악
- **"책 카드 만든 거 어디에 올리세요? 인스타 / 단톡방 / 둘 다?"** — viral 채널 우선순위 정확히
- **"인용구 다시 본 적 있어요? 언제 어떻게?"** — Readwise식 알림 가치 검증
- **"친구가 만든 카드 본 적 있어요? 어디서?"** — viral 트래픽이 어느 채널에서 발생하는지

다음 질문은 응답 편향 위험 — 빼거나 후순위로:
- "(컨셉 설명 후) 어때요?" → 너무 빨리 솔루션 보여줌. 실제 인터뷰에서는 솔루션 설명을 인터뷰 마지막 5분으로 미뤄야 함. 그 전엔 **행동·페인만** 캐물어야 응답 편향 회피
