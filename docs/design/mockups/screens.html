  • 중간 책 못 골라도 저장 가능(BOOK_UNRESOLVED) — 시트에 [ISBN 직접 등록] 출구
  • 낮음 권한 흐름 0 (카메라·사진 권한 요청 안 함 — 내장 OCR 없음)

신규

  • quotes 테이블(book_id nullable, manual_book_text, text CHECK 1~2000, source manual/clipboard, moods text[]), quote_repository/quote.dart/quote_providers/createQuoteController/quote_outbox. pubspec: shared_preferences·connectivity_plus

주요 인터랙션

  • 진입 직후: fallback 팔레트로 즉시 렌더(빈 화면 0초) → 표지 fetch 도착 시 cross-fade → 팔레트 추출 완료 시 색만 cross-fade. 추천 템플릿이 선택된 상태로 시작
  • "색 변경 UI"를 따로 안 만듦 — 표지 5스와치 줄이 곧 컬러 패널. 탭하면 현재 템플릿의 색 슬롯에 적용. 흑백 표지는 채도<10 폴백
  • 인용구 텍스트 박스 = "한 번 탭" → 인라인 핸들(이동/크기). 더블탭 숨김 금지. 이동은 상/중/하 3지점 스냅 앵커(템플릿 미관 보호)
  • 언두 필수(Mojo가 없어서 욕먹은 지점) — 최소 20단계, 비율·템플릿 전환까지 포함. 비율 전환 시 텍스트 위치 재매핑(상대좌표 0~1 + clamp)
  • 편집 상태(템플릿·팔레트·앵커·비율·undo 스택) 디버운스 영속화 → 재진입 "이어 만들기". 인용구 본문은 DB라 안전 → 뒤로 시 폐기 확인 불필요

상태

  • 높음 PNG 생성 실패(OOM) → Toast + 디자인 전부 유지 → 재시도 시 1080→720 폴백
  • 높음 폰트 로드 완료를 캡처 전에 보장(endOfFrame + 미로드 시 1회 재시도) — 안 그러면 첫 캡처가 □□□로 인스타에 박힘. NotoSerifKR은 앱 번들 asset(런타임 다운로드 X)
  • 높음 긴 인용구 auto-fit 한계 — 잘린 채 export 절대 금지. 최소 폰트 도달 후 넘치면 비율 자동 추천 → 그래도 안 되면 명시적 경고
  • 높음 표지 색 vs 텍스트 대비 부족 → ensureContrast(WCAG AA 4.5:1) + T4 그라데이션 overlay. 깨지면 가독성 0 → 스냅샷+대비 테스트 필수
  • 중간 표지 없는 책 → T1/T3/T5 정상, T2/T4는 fallback 팔레트. T4(표지발췌) 처리 미결: 비활성화 vs 단색 degrade — 권고 비활성화

미결

  • 텍스트 위치 앵커(상/중/하) V1에 넣을지 — 넣으면 템플릿 좌표 모델 변경 수반. 권고: V1은 폰트 크기 ±·정렬만, 앵커는 V1.5
  • 표지 없는 책에서 T4 — 권고: 비활성화(썸네일 회색 + "표지가 필요해요")

재사용

  • 인프라 거의 완비: ExtractedPalette·fallbackPalettes·getQuoteFontSize·CardRatio·AppCardSize·AppWatermark·palette_generator·color-extraction.md. 신규: card_template.dart(sealed×5)·quote_card.dart·palette_service·card_renderer·cards 테이블. pubspec: share_plus·path_provider

  • 중간 카카오톡/인스타 미설치 → OS 시트에 자동으로 안 뜸(무해). 전용 버튼이면 canLaunchUrl 체크 후 안내, 크래시 금지
  • 중간 네트워크 끊겨도 공유 자체는 됨(PNG 로컬, OS 시트 로컬). cards 히스토리 INSERT만 큐/skip → 사용자엔 "성공"
  • 낮음 공유 시트 취소 = 에러 아님, Toast 없음, 에디터 유지, 히스토리 저장 안 함

주요 인터랙션

  • deep link 수신 → deep_link_handler 분기: auth code면 getSessionFromUrl(기존), 아니면 router.go('/book/$id?from=$from')
  • 미로그인 → router redirect로 /auth/login?from=... + handler가 pending deep link 보관 → 로그인 후 소비 → 책 상세 복귀
  • "내 서재에 담기" → addToLibrary(idempotent) → "서재에 담았어요 ✓" + "이 책에서 인용구 모으기" 권유
  • 같은 deep link를 단톡방 스크롤하며 여러 번 탭 → idempotent, Toast 폭탄 방지

상태

  • 높음 미로그인 진입(현재 미구현) → read-only 먼저 + payload 보존 후 로그인→복귀
  • 높음 콜드스타트+미로그인+deep link → /splash→로그인 게이트→보존된 link 소비. 가입이면 "프로필 생성 중" 짧은 대기 후
  • 중간 그 책/인용 삭제됨(PGRST116) → "더 이상 볼 수 없어요" + [홈으로]/[책 검색]. 비공개·차단 인용은 책 정보만
  • 중간 이미 서재에 있는 책(23505) → "이미 서재에 있어요" 정보성 Toast(에러 아님)
  • 중간 deep link 무한 루프 → 앱당 1회 consume 후 클리어, redirect 최대 1홉
  • 중간 앱 미설치 → 커스텀 스킴은 fallback 없음. 웹 뷰어(Universal/App Link + 도메인 + assetlinks)는 V1.5. V1은 "앱 있는 사람끼리만" 한계 + 안내 카피

주요 인터랙션

  • 서재 헤더 세그먼트: "책" ↔ "인용구" — 전환 시 스크롤 위치 각자 유지(StatefulShellRoute state 보존)
  • 무드 칩 = 단일 선택 필터(개수 표시). 인용구 카드 탭 → 인라인 확장(전체 텍스트 + 메모 편집 + 액션: 카드 만들기 / 수정 / 무드 변경 / 공유 / 삭제)
  • 무드 변경 = 카드에서 직접 칩 토글(Readwise 인라인 태깅) — 저장 즉시 낙관적. 인용 저장 직후 토스트의 "무드 추가?" 칩도 같은 경로
  • 스와이프 삭제 X(우발 삭제 우려) — 확장 후 [삭제] 버튼 + undo SnackBar 5s
  • 정렬: 최근순(기본) / 책별 그룹 / 페이지순. 페이지네이션 = cursor-after(created_at+id), 15개 — offset 금지(DECISIONS 2026-05-10)

상태

  • 중간 빈: "아직 인용구가 없어요…" + [＋ 인용구 추가]. 특정 무드 0개: "이 무드의 인용구가 아직 없어요" + [전체 보기]
  • 중간 오프라인: 마지막 캐시 표시(stale-while-revalidate) + 배너. 동기화 대기 인용구는 "동기화 대기" 뱃지 + 책 매칭 실패 시 "책 정보 필요" 액션
  • 중간 책 없는 인용구(BOOK_UNRESOLVED): 표지 자리 placeholder + "책 연결하기" 인라인 액션
